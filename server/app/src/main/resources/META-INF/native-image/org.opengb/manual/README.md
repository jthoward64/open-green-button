# Hand-maintained native-image metadata

`reflect-config.json` here is **not** generated by `scripts/generate-native-metadata.sh` (which
writes `../from-tests/` and `../from-app/`). native-image unions every subdirectory under
`META-INF/native-image/`, so entries here are merged with the agent-generated ones.

The tracing agent can't capture this reliably, so it is registered here by hand:

1. **The Hoplite config data classes** (`org.opengb.config.*`, `org.opengb.utility.*`). Hoplite
   decodes these via kotlin-reflect, using each data class's synthetic `<init>$default` constructor
   to fill in default arguments. The agent records the primary constructor but not that synthetic
   one, so a config that relies on a default value fails at runtime with
   `Could not compute caller for function: … constructor`. `allDeclaredConstructors` registers the
   synthetic constructor too. Add new config/profile data classes here when you introduce them.
